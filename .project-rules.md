# Project Rules & Project Memory

## 1. Vietnamese Currency Formatting
- **ALWAYS strictly use `,` (comma)** for thousands separators based on this project's standard format.
- Example: `1,000,000` (One million VNĐ), NOT `1.000.000`.
- When using Ant Design `InputNumber`, apply the following formatter and parser:
  ```ts
  formatter={(val) => `${val}`.replace(/\B(?=(\d{3})+(?!\d))/g, ',')}
  parser={(val) => val!.replace(/\$\s?|(,*)/g, '') as any}
  ```
- Any money format utility should leverage `.toLocaleString('en-US')` perfectly to match the comma format.

## 2. Exact Reporting Column Terminology (UI & Labels)
- ALWAYS strictly use exact headers / column terminology provided by legacy Excel report sheets or user screenshots.
- **DO NOT** invent verbose equivalents (e.g. changing `Tiền mặt` to `Doanh thu TM Quầy`, or `Chi TM` to `Chi phí TM Quầy`, `Thực nhận` to `Tiền mặt bàn giao thực tế`).
- **Standard 'Báo cáo kết ca' vocabulary:** 
  - `Ngày`, `CA`
  - `FNET`, `GCP`, `MOMO`
  - `Tiền mặt`, `Chi TM`, `Sau chi`
  - `Thực nhận`, `Chênh lệch`, `Tổng thực nhận` (hoặc `Tổng thực nhận tính gộp`)
- Adhering to the exact words prevents user confusion comparing the software against physical sổ sách (accounting books).
